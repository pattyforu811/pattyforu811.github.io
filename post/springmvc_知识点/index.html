/和/* 配置问题

为什么直接访问html访问不到？

  1. 服务器的web.xml中有个默认配置

    1
2
3
4

    <servlet-mapping>
     <servlet-name>default</servlet-name>
     <url-pattern>/</url-pattern>（父） 用来处理静态资源
</servlet-mapping>

    如果我们项目中有html，交给tomcat进行处理的话，那么就能找到对应的资源

  2. 我们项目中的前端控制器

    1

    <url-pattern>/</url-pattern>（子）

    子和父都有相同的配置情况下，最终指定的子，那么不会请求到tomcat中去，前端控制器会去找那个方法上的RequestMapping 注解上声明的 index.html

为什么jsp可以访问到

  1. tomcat 的 web.xml配置

    1
2
3
4

    <servlet-mapping>
    <servlet-name>jsp</servlet-name>
    <url-pattern>*.jsp</url-pattern>（父）
</servlet-mapping>

  2. 项目配置

    1

    <url-pattern>/</url-pattern>(子)

    子没有覆盖父，子处理不了的，交给父进行处理，父找到相应的资源直接进行返回
    而如果项目配置成这样： <url-pattern>*.jsp</url-pattern>(子)
    子和父都有相同的配置情况下，最终指定的子，那么不会请求到tomcat中去， 前端控制器会去找哪个方法上的RequestMapping注解上声明的 index.jsp

请求参数绑定

需要绑定什么参数就直接写在方法的参数(括号)里

  • 当绑定参数普通属性,前台直接写属性名
  • 当绑定的参数是实体且属性是普通属性,前台直接写属性名
  • 当绑定的参数是实体且属性是另一个实体B,前台直接写实体B.属性名(属性名是实体B的属性)
  • 当绑定的参数是集合类型时:
    1. List[User] list;
      list[0].username
      list[0].userpsd
    2. Map <String,User> map;
      map[‘one’].username
      map[‘one’].userpsd
      one是设置的键值对的key

package io.pattyforu.utils.converter;

import org.springframework.core.convert.converter.Converter;
import java.text.DateFormat;
import java.text.ParseException;
import java.text.SimpleDateFormat;
import java.util.Date;

/**
 * 字符串转日期的类型转换器
 */
public class StringToDateConverter implements Converter<String, Date> {

    /**
     * @param s 传入进来的字符串
     * @return
     */
    @Override
    public Date convert(String s) {
        if (s == null) {
            throw new RuntimeException("请输入你的数据");
        }
        DateFormat dataFormat = new SimpleDateFormat("yyyy-MM-dd");

        try {
            return dataFormat.parse(s);
        } catch (ParseException e) {
            e.printStackTrace();
            throw new RuntimeException("日期转换错误");
        }
    }
}

配置静态资源

DispatcherServlet会拦截到所有的资源，导致一个问题就是静态资源（img、css、js）也会被拦截到，从而不能被使用。
解决问题就是需要配置静态资源不进行拦截，在springmvc.xml配置文件添加如下配置:

<!-- 设置静态资源不过滤 -->
<!--1. location元素表示webapp目录下的包下的所有文件-->
<!--2. mapping元素表示以/static开头的所有请求路径，如/static/a 或者/static/a-->
<mvc:resources location="/css/" mapping="/css/**"/> <!-- 样式 -->
<mvc:resources location="/images/" mapping="/images/**"/> <!-- 图片 -->
<mvc:resources location="/js/" mapping="/js/**"/> <!-- javascript -->

常用注解

@RequestMapping

用来处理请求地址映射的注解，可用于类或方法上。用于类上，表示类中的所有响应请求的方法都是以该地址作为父路径

属性:

  • value/path: 指定请求的实际地址，指定的地址可以是URI Template 模式
  • method： 指定请求的method类型， GET、POST、PUT、DELETE等(如RequestMethod.GET),多个的话用{}括起来
  • params: 指定request中必须包含某些参数值(“username”),(“username=aaa”)，才让该方法处理。(跟绑定参数不一样)
  • headers: 指定request中必须包含某些指定的header值，才能让该方法处理请求
  • consumes： 指定处理请求的提交内容类型（Content-Type），例如application/json, text/html;

  • produces：告诉浏览器返回的内容类型是什么， 细节:

  • 路径可以不编写 / 表示应用的根目录开始

  • ${ pageContext.request.contextPath }也可以省略不写，但是开始路径上不能写 /

@RequestParam

位置:
　　写在方法形参的前面
作用：
　　把request请求中指定名称的参数给控制器中的形参赋值。
属性：
　　value：请求参数中的名称。
　　required：请求参数中是否必须提供此参数。
　　　　　　　默认值：true。表示必须提供，如果不提供将报错。
　　　　　　　值：false。表示不是必须提供

@RequestBody

位置:
　　写在方法形参的前面
作用：
　　用于获取请求体内容。直接使用得到是 key=value&key=value…结构的数据。
　　get 请求方式不适用, get请求的参数在url里,没有请求体。
属性：
　　required：是否必须有请求体。
　　　　　　　默认值：true。当取值为 true 时,get 请求方式会报错。
　　　　　　　值：false。表示不是必须提供, get 请求得到是 null。

@PathVaribale

位置:
　　写在方法形参的前面
作用：
　　用于绑定 url 中的占位符(获取url占位符的值)。
　　例如：请求 url 中 /delete/{id}，这个{id}就是 url 占位符。
　　url 支持占位符是 spring3.0 之后加入的。
属性：
　　value：用于指定 url 中占位符名称。
　　required：是否必须提供占位符。

@ModelAttribute 注解

位置:
　　出现在方法和参数上 作用：
　　出现在方法上：表示当前方法会在控制器方法(要访问的方法)执行前线执行。
　　出现在参数上：获取指定的数据给参数赋值。
属性：
在参数上
　　value：用于获取数据的 key。key 可以是 POJO 的属性名称，也可以是 map 结构的 key。
　　应用场景：
　　当表单提交数据不是完整的实体类数据时，为保证没有提交数据的字段可以使用数据库对象原来的数据
　　例如：我们在编辑一个用户时，用户有一个创建信息字段，该字段的值是不允许被修改的。 　　　　　表单是肯定没有此字段的内容,但要把该字段内容置为指定数据如 ll，此时就可以使用此注解解决

@Controller
//映射器 一级目录
@RequestMapping("/springmvc")
//把数据存入到session域对象中
@SessionAttributes(value = {"username", "password","age"}, types = {String.class,Integer.class})
public class HelloController {
    /**
     * 把数据存入 SessionAttribute
     * 使用sevlet原生的session 对象， 耦合度太高， 不推荐
     * Model 是 spring 提供的一个接口，该接口有一个实现类 ExtendedModelMap
     * 该类继承了 ModelMap，而 ModelMap 就是 LinkedHashMap 子类
     */
    @RequestMapping("/testPut")
    private String useSessionAttribute(Model model) {

		//底层存储到request域对象中
        model.addAttribute("username", "名字");
        model.addAttribute("password", "psw");
        model.addAttribute("age", "20");
        return "success";
    }

    /**
     * 从session中取值
     * @param model
     * @return
     */
    @RequestMapping("/testGet")
    public String testGet(ModelMap model) {
        System.out.println(model.get("username") + ";" + model.get("password") + ";" + model.get("age"));
        return "success";
    }

    /**
     * 清除值
     * @param sessionStatus
     * @return
     */
    @RequestMapping("/testClean")
    public String complete(SessionStatus sessionStatus){
        sessionStatus.setComplete();
        return "success";
    }
}

总结

model.addAttribute();方法可以把数据存储到request域中
@SessionAttributes() 注解可以指定把什么数据存储到session域中

只调用model.addAttribute();方法, 只有request中有值,
有@SessionAttributes()注解, request和session都有值,
当调用sessionStatus.setComplete();方法清除session中的值时, 返回success页面, request域中还有值, 但是session域中没有值, 再一次请求取值, 两个域中都没有值了
