import org.aspectj.lang.ProceedingJoinPoint;

/**
 * 用于记录日志的通知类
 *
 */
public class LogUtils {
    
    /*
     用于输出日志记录，想要在切入点方法执行之前执行
     */
    public void logStart(){
        System.out.println("LogUtils的logStart方法开始记录日志了");
    }
    /**
     * 后置,想要在切入点方法执行之后执行
     */
    public void logEnd(){
        System.out.println("LogUtils的logEnd方法开始记录日志了");
    }
    
    /**
     * 异常，想要在切入点方法发生异常之后执行
     */
    public void logException(){
        System.out.println("LogUtils的logException方法开始记录日志了");
    }
    
    /**
     * 最终，想要在切入点是否正常执行都会在最后执行
     */
    public void logFianl(){
        System.out.println("LogUtils的logFianl方法开始记录日志了");
    }
    /**
     * 环绕通知
     * <!-- 问题: 当配置环绕通知之后，切入点方法没有执行，而通知方法执行了
               解决:
                    ProceedingJoinPoint:接口，proceed()明确调用切入点方法，可以做为环绕通知的参数
                    在程序执行时，spring框架会提供接口的实现类给我们使用
     */
    public Object logAround(ProceedingJoinPoint pjp){//动态代理的invoke方法一样
        Object obj = null;
        try {
            Object[] objs = pjp.getArgs();//得到方法执行的所需参数
             System.out.println("前置");
             obj = pjp.proceed(objs);//明确调用切入点（业务层）方法
             System.out.println("后置");
            return obj;
        } catch (Throwable e) {
            System.out.println("异常");
            throw new RuntimeException();
        }finally {
            System.out.println("最终");
        }
        
    }

    <!-- aop配置 -->
<aop:config>
<!-- pointcut:用于指定切入点表达式，该表达式的含义代表
	 对业务层service中的哪些方法进行增强 execution(表达式) -->
	<aop:pointcut expression="execution(* com.pattyforu.service.impl.*.*(..))" id="pt"/>
<!-- 配置切面 : 指定通知和切入点之间的关系 -->
<aop:aspect id="logAdivce" ref="logUtils">
	<!-- 配置前置通知：
		method属性：用于指定logUtils类中哪个方法作为前置通知-->
	   <!-- <aop:before method="logStart"  pointcut-ref="pt"/> -->
	   <!-- 配置后置通知 -->
	   <!-- <aop:after-returning method="logEnd" pointcut-ref="pt"/> -->
	   <!-- 配置异常通知 -->
	   <!-- <aop:after-throwing method="logException" pointcut-ref="pt"/> -->
	   <!-- 最终通知 -->
	   <!-- <aop:after method="logFianl" pointcut-ref="pt"/> -->
	   <!-- 环绕通知 -->
	   <aop:around method="logAround" pointcut-ref="pt"/>
</aop:aspect>
</aop:config>

表达式的写法：

  • 方法定义：public void saveUser() {}
  • 访问权限修饰符 返回值 包名.类名.方法名(参数列表)
  • 表达式写法：public void com.pattyforu.service.impl.UserServiceImpl.saveUser()

  • 原始表达式 到 全通配表达式的过程：

    1. 访问权限修饰符可以省略
      void com.pattyforu.service.impl.UserServiceImpl.saveUser()
    2. 返回值类型可以使用通配符 *
      `* com.pattyforu.service.impl.UserServiceImpl.saveUser()
    3. 包名可以使用通配符 ，表示任意包，但是又几级包就有些几个*.
      * *.*.*.*.UserServiceImpl.saveUser()

    4. ..表示当前包以及子包
      * *..UserServiceImpl.saveUser()

    5. 类名和方法名都可以使用通配符进行匹配
      * *..*.*()

    6. 方法的参数就可以使用.. 代表任意参数
      * *..*.*(..)

    7. 全通配写法：
      * *..*.*(..)

建议配置：

切到业务层实现类下的所有方法
　　* com.pattyforu.service.impl.*.*(..)

知识点

连接点(joinpoint)：目标对象中,所有可以增强的方法(相当于所有方法)

切入点(pointcut)：目标对象中,所有已经增强的方法

通知(advice)：增强的代码的类

目标对象(target)：被代理的对象(需要增强的方法的类)

织入(weave)：将通知织入目标对象

代理(proxy)：将通知织入目标对象之后,形成的代理对象(代理对象就是织入通知后的目标对象)

切面(aspect)：切入点+通知
